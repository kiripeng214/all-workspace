# PRD: 体重字段自动拼接单位

## 1. 背景与动机

- 目前体重字段需要用户手动输入 "15kg" 或 "500g"，格式不统一、体验差
- kg/g 是固定单位，不应让用户手动输入，应由前端自动拼接
- 避免因格式不一致（"15kg" vs "15 kg" vs "15"）导致数据混乱

## 2. 需求描述

### 2.1 功能列表

| # | 功能 | 优先级 | 说明 |
|---|------|--------|------|
| 1 | 体重输入拆为数字框 + 单位选择器 | P0 | 左：数字输入框 type=digit；右：kg/g 选择器 |
| 2 | 提交时自动拼接 | P0 | 空值跳过，有值拼为 "数字+单位"，如 "15kg" |
| 3 | 编辑时自动解析 | P0 | 已有数据 "15kg" → 数字=15、单位=kg 分别填入对应控件 |
| 4 | 模型和 API 不改 | P0 | 后端仍存 string，前端只改交互层 |

### 2.2 用户流程

```
创建宠物 / 编辑宠物
  → 体重字段：左输入数字，右选 kg/g
  → 提交：自动拼接 "15kg"
  → 详情页显示：⚖️ 15kg
  → 再次编辑：自动拆回 15 + kg
```

## 3. 非功能性需求

- 数字输入框使用 `type="digit"`，移动端自动弹出数字键盘
- 单位选择器使用原生 picker

## 4. 验收标准

- [ ] 创建宠物页：体重字段显示为数字输入框 + kg/g 选择器
- [ ] 只输入数字不选单位，默认 kg
- [ ] 提交后后端存的是 "15kg" 格式（带单位）
- [ ] 编辑已有宠物：体重数字和单位正确回显到对应控件
- [ ] 空体重不拼接，提交空字符串
- [ ] 类型检查通过：`vue-tsc --noEmit`
- [ ] 测试通过：`vitest run`
