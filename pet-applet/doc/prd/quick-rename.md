# PRD: 宠物名称快速编辑

## 1. 背景与动机

- 当前修改宠物名称需要进入详情页 → 点击「编辑」→ 跳转到完整编辑页 → 修改名称 → 保存 → 返回
- 操作路径过长，切换页面多，只是改个名字却要打开整个编辑表单
- 期望在详情页直接点击名字修改，无需跳转

## 2. 需求描述

### 2.1 功能列表

| # | 功能 | 优先级 | 说明 |
|---|------|--------|------|
| 1 | 详情页点击宠物名字弹出改名弹窗 | P0 | 直接点击名字即可编辑 |
| 2 | 改名弹窗含名称输入框和确认按钮 | P0 | 当前名称预填，可修改后保存 |
| 3 | 保存后页面自动刷新 | P0 | 显示新名称 |

### 2.2 用户流程

```
宠物列表 → 点击卡片进详情 → 点击宠物名字 → 弹出改名弹窗（预填当前名称）
    ↘ 确认 → 调用 API 保存 → 弹窗关闭 → 页面刷新
    ↘ 取消/点击蒙层 → 弹窗关闭，无变化
```

## 3. 非功能性需求

- 输入框不能为空，为空时点确认给出提示
- 保存失败时给出 toast 错误提示
- 加载状态（保存中）禁用按钮防止重复提交
- 弹窗使用 uni-popup 组件

## 4. 验收标准

- [ ] 详情页点击宠物名字弹出改名弹窗
- [ ] 弹窗中正确显示当前宠物名称
- [ ] 修改名称后点击确认，名称更新成功
- [ ] 名称不能为空，为空时提示
- [ ] 保存成功后页面刷新，显示新名称
- [ ] 点击取消或蒙层，弹窗关闭，名称不变
