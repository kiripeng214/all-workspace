# PRD: 详情页点击名字快速改姓名

## 1. 背景与动机

- 当前修改宠物姓名需要：详情页 → 点「编辑」→ 跳转编辑页 → 修改 → 保存 → 返回
- 只是改个名字要跳转整个页面，操作路径太长
- 期望在详情页直接点击姓名修改，不跳转

## 2. 需求描述

### 2.1 功能列表

| # | 功能 | 优先级 | 说明 |
|---|------|--------|------|
| 1 | 详情页点击宠物姓名弹出编辑弹窗 | P0 | 当前姓名预填 |
| 2 | 弹窗内修改后提交后端保存 | P0 | 调用 PUT /pets/:id |
| 3 | 保存后页面刷新显示新姓名 | P0 | 无需手动刷新 |

### 2.2 用户流程

```
宠物列表 → 点击进详情 → 点击姓名 → 弹窗（预填当前姓名）
    ↘ 修改后点确认 → 保存 → 弹窗关 → 页面刷新
    ↘ 取消/点蒙层 → 弹窗关，不变
```

## 3. 非功能性需求

- 姓名为空时提示，不能保存
- 保存失败 toast 提示
- 保存中禁用按钮防重复提交
- 弹窗使用 uni-popup，与记录喂养弹窗风格一致

## 4. 验收标准

- [ ] 详情页点击姓名弹出编辑弹窗
- [ ] 弹窗预填当前姓名
- [ ] 修改后确认，姓名更新成功
- [ ] 姓名为空时提示
- [ ] 保存后页面刷新
- [ ] 取消/点蒙层弹窗关闭，不修改
