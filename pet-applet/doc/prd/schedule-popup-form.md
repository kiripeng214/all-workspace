# PRD: 喂养计划表单改为弹出框

## 1. 背景与动机

- 当前喂养计划页面表单始终内联显示在列表下方，占用固定页面空间
- 当只有 1-2 条计划时，页面整体内容稀疏，表单占据大量空间影响浏览体验
- 改为弹出框后，列表更加简洁，用户聚焦于数据浏览，仅在需要时通过点击触发表单

## 2. 需求描述

### 2.1 功能列表

| # | 功能 | 优先级 | 说明 |
|---|------|--------|------|
| 1 | 列表页始终显示计划列表，不显示表单 | P0 | 删除内联表单，列表下方不再有表单区域 |
| 2 | 点击"添加计划"按钮弹出表单 | P0 | 新增一个浮动添加按钮（FAB），点击弹出表单覆盖层 |
| 3 | 点击"编辑"操作弹出已有数据的表单 | P0 | 编辑时表单预填当前计划数据 |
| 4 | 弹出框支持取消关闭 | P0 | 点击遮罩层或取消按钮关闭 |
| 5 | 表单在弹出框居中显示 | P1 | 淡白色遮罩 + 居中白色卡片 |

### 2.2 用户流程

```
进入喂养计划列表
  → 查看已有计划列表
  → 点"添加计划" FAB 按钮 → 弹出表单（空白）
     → 填写时间/食物/分量 → 提交 → 表单关闭 → 列表刷新
     → 点取消/遮罩层 → 表单关闭
  → 点计划项的"编辑" → 弹出表单（已填当前数据）
     → 修改 → 提交 → 表单关闭 → 列表刷新
  → 点计划项的"删除" → 确认弹窗 → 删除 → 列表刷新
```

## 3. 非功能性需求

- 动画过渡：弹出框出现时有淡入效果
- 遮罩层：点击遮罩层可关闭表单
- 兼容 H5 和微信小程序

## 4. 验收标准

- [ ] 进入页面，只显示计划列表，列表下方没有表单区域
- [ ] 列表末尾有个"+ 添加计划"样式的按钮/入口
- [ ] 点"+ 添加计划"弹出遮罩 + 白色表单卡片，表单内容为空
- [ ] 表单弹出框在屏幕居中，白色背景，圆角 16rpx
- [ ] 遮罩层为半透明黑色（rgba(0,0,0,0.4)），覆盖全屏
- [ ] 表单有：时间选择器、食物输入框、分量输入框、取消/提交按钮
- [ ] 点"取消"按钮关闭弹出框
- [ ] 点遮罩层关闭弹出框
- [ ] 点计划项的"编辑"弹出表单，表单预填该计划的时间/食物/分量
- [ ] 提交成功后弹出框关闭，列表刷新
- [ ] 类型检查通过：`vue-tsc --noEmit`
- [ ] 测试通过：`vitest run`
