# PRD: 宠物知识库 (RAG + chromem-go + LLM)

## 1. 背景与动机

- 用户在养宠过程中遇到问题（"金毛掉毛怎么办"、"猫不吃东西"）需要即时获取知识
- 直接在宠物详情页内嵌入知识库，根据宠物品种自动推荐相关知识
- RAG 架构确保回答基于可靠的知识库内容，而非 LLM 凭空生成

## 2. 需求描述

### 2.1 功能列表

| # | 功能 | 优先级 | 说明 |
|---|------|--------|------|
| 1 | 后端 RAG API | P0 | chromem-go 向量存储 + 相似度搜索 |
| 2 | 知识库种子数据 | P0 | 宠物饲养、健康、行为等常见知识条目 |
| 3 | LLM 生成回答 | P0 | 基于检索到的上下文，用 LLM 生成自然语言回答 |
| 4 | 前端知识页面 | P0 | 宠物详情页新增"知识"入口，展示相关知识 |
| 5 | 按品种自动匹配 | P1 | 根据宠物品种自动加载相关知识 |

### 2.2 用户流程

```
进入宠物详情页 → 点击"知识"按钮
  → 调用 RAG API（传品种/问题）
  → 后端：chromem-go 向量检索 → 找到最相关的知识条目
  → LLM 基于检索结果生成自然回答
  → 前端展示：问题标题 + AI 回答 + 参考来源
```

## 3. 验收标准

- [ ] 后端 `/api/knowledge/search` 接口可检索知识库
- [ ] chromem-go 成功构建向量索引并返回相似结果
- [ ] LLM 基于检索上下文生成通顺回答
- [ ] 前端展示知识卡片
- [ ] 后端编译通过：`go vet ./...` + `go test ./...`
- [ ] 前端类型检查通过：`vue-tsc --noEmit`
